>>> You are a strict coding teacher. Explain what a function is.
In programming, a function is a reusable block of code that performs a specific task. It is a way to modularize
your code, meaning that you can break down your program into smaller, more manageable parts. Functions are used to
perform specific tasks, such as calculations, data manipulation, or input validation.

A function typically has a name, and it takes in one or more inputs, which are called parameters. The inputs can
be of different types, such as integers, strings, or objects. Once the function is called, it executes the code
block inside it, and then returns a value. This returned value can be used later in the program.

Functions are essential in programming, as they make code more readable, maintainable, and reusable. They also
help to avoid code duplication and make it easier to test and debug code.

Here is an example of a simple Python function that takes in two numbers as input and returns their sum:

```python
def add_numbers(x, y):
    return x + y
```

To call this function, you would write:

```python end
result = add_numbers(3, 4)
print(result)  # Output: 7
```

In this example, the `add_numbers` function takes in two numbers, `x` and `y`, and returns their sum. We call the
function with the values `3` and `4`, and it returns the value `7`. We then store this value in a variable called
`result` and print it to the console.

In summary, functions are an essential part of programming, and they help to make code more readable,
maintainable, and reusable. They are used to perform specific tasks and can be called multiple times throughout a
program.

📝 Day 1 — Theory Notes
1️⃣ What is Prompt Engineering?

Prompt engineering is the practice of designing and structuring instructions given to a Large Language 
Model (LLM) in order to control its output. It involves carefully choosing wording, role instructions,
formatting rules, and constraints so the model produces accurate, structured, and useful responses.

2️⃣ Why Does Wording Matter?

Wording matters because LLMs generate responses based on patterns in text. Small changes in instructions 
can significantly affect:

Depth of explanation

Tone and personality

Structure of output

Level of detail

Target audience

Better wording leads to more controlled and predictable results.
